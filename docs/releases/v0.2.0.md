# opencode-agenthub v0.2.0

Released: 2026-04-04

## Highlights

- Adds a new authoring lane so users can create and refine Agent Hub skills and agents directly with built-in guidance.
- Upgrades the built-in coding workflow with wave-aware planning, proactive verification, and branch-finishing support.
- Keeps HR Office scoped to staffing and staging while making normal auto installs more capable out of the box.

## Detailed changes

### Coding workflow upgrades

- Upgraded the built-in planning and execution skills used by the coding showcase so they can express wave-structured plans and verification checkpoints more explicitly.
- Added built-in `agenthub-proactive-verify` and `agenthub-prepare-pr` skills for verification-before-completion and branch-based finishing flows.
- Updated the built-in `demo-coding-team` assets so the delivery lead bundle, soul, and support docs align with the stronger workflow.

### Authoring lane

- Added `write-skill` for drafting portable `SKILL.md` assets with required frontmatter and focused workflow guidance.
- Added `write-agent` for creating soul + bundle + optional profile assets outside the HR staffing lane.
- Added `refine-hub-asset` for safely editing existing souls, bundles, profiles, instructions, and skills with asset-specific verification guidance.
- Added `validateSkillFrontmatter()` as a hub-local validator for skill frontmatter parsing and warnings.

### Built-in asset registration and validation

- Added an `authoring-lane-guide` instruction and registered the new authoring assets for auto-mode installs.
- Kept HR Office built-in scope focused on HR-only assets instead of automatically inheriting the authoring lane.
- Added regression coverage for the expanded auto manifest and verified that install / compose flows still pass.
