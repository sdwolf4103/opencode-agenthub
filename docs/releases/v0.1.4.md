# opencode-agenthub v0.1.4

Released: TBD

## Highlights

- Adds `agenthub status` so workspace runtime state is inspectable without reading generated files by hand.
- Makes plugin and OMO runtime boundaries visible, including local plugin bridge status and OMO baseline inheritance.
- Upgrades `agenthub doctor` into a category-aware troubleshooting entrypoint with linked remediation docs.

## Detailed changes

### Runtime visibility

- Added `agenthub status`, `agenthub status --short`, and `agenthub status --json` for composed runtime inspection.
- Compose summaries now surface runtime source and point users toward `agenthub status` for deeper inspection.

### Plugin and OMO boundary visibility

- Runtime status and doctor output now show whether local filesystem plugins were bridged into the composed runtime.
- Added explicit support for `settings.json -> localPlugins.bridge` so users can opt out of local plugin copying.
- Runtime status and doctor output now show whether Agent Hub is inheriting a global `oh-my-opencode.json` baseline or isolating from it through `omoBaseline = "ignore"`.

### Doctor and troubleshooting

- Added doctor categories for `home`, `environment`, `workspace`, and `plugin`.
- Added structured check ids, remediation text, and linked troubleshooting docs.
- Added short troubleshooting guides for environment setup, compose failures, plugin degraded mode, guard/OMO conflicts, and model configuration.
