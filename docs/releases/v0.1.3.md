# opencode-agenthub v0.1.3

Released: 2026-03-30

## Highlights

- Restores `K-Dense-AI/claude-scientific-skills` as a default HR bootstrap source for new HR homes.
- Brings HR staffing guidance back in line with the intended operating model for larger teams: usually one to two primary agents plus subagents, not many primary agents.
- Simplifies HR model conversations so confirmation follows what the user's current opencode environment can actually run.
- Makes the fallback for user-insisted but unverified models non-argumentative: leave `agent.model` blank, let opencode use its default model, and tell the operator where to override it later.

## Detailed changes

### HR bootstrap defaults

- Added `K-Dense-AI/claude-scientific-skills` to the default GitHub sources written into fresh HR Office `hr-config.json` files.
- Updated README guidance so the default-source documentation matches the actual bootstrap behavior.

### HR staffing and review flow

- Added back the large-team advisory in HR staffing/review prompts so teams larger than four agents are guided toward one to two primary agents with the rest staged as subagents.
- Kept this as guidance rather than a hard gate.

### HR model confirmation

- Removed HR prompt guidance that treated the synced model inventory catalog as the conversational source of truth.
- Updated HR prompts and worker docs so model confirmation follows the user's current opencode environment instead.
- Removed proactive HR questions about whether the promoted profile should become the default profile.

### Unverified model fallback

- When a user insists on a specific model that cannot be verified in the current opencode environment, HR now stages no `agent.model` override instead of blocking, arguing, or silently swapping models.
- The handoff guidance now tells operators to update the staged bundle JSON directly or use `agenthub doctor` after promote if they still want to force a specific model later.
